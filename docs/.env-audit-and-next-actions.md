# Аудит `.env` та наступні операційні дії для індексації

Цей документ створено, оскільки всі чеклісти в `tasks.md` на задачу руйнування перераховані, а реальна перевірка індексації через Google та Bing залишається заблокованою відсутністю облікових даних.

## Чек-ліст `.env`

| Змінна | Призначення | Статус |
|--------|-------------|--------|
| `GSC_SITE_URL` | Canonical-домен для Google Search Console | Не заповнено |
| `GSC_KEY_PATH` | Шлях до сервісного ключа GSC у форматі JSON | Не заповнено |
| `BING_SITE_URL` | Домен для Bing Webmaster Tools | Не заповнено |
| `BING_WEBMASTER_API_KEY` | API-ключ Bing Webmaster | Не заповнено |

## Передумови для запуску

1. Створити сервісний акаунт у Google Cloud і увімкнути `Web Search API`/Search Console API.
2. Завантажити JSON-ключ сервісного акаунта і розмістити за шляхом, вказаним у `GSC_KEY_PATH`.
3. Додати сайт у Bing Webmaster Tools і отримати API-ключ.
4. Заповнити `.env` реальними значеннями.

## Команда перевірки

```bash
bash scripts/indexing-check.sh
```

## Наступні діі

1. Заповнити `.env` згідно з прикладом у `docs/next-steps.md`.
2. Перезапустити `bash scripts/indexing-check.sh`.
3. Зафіксувати результат у `docs/indexing-protocol.md`.
