# Runbook: Автоматизована реєстрація/підтвердження соцмережних профілів

Цей документ фіксує порядок створення та підтвердження акаунтів соцмереж для проєкту BizTool Insight. Це розширює `docs/social-profile-runbook.md` та доповнює `docs/social-setup-docs.md`.

## Мета
- Створити/підтвердити акаунти в платформах: X/Twitter, Facebook Page, Instagram Business, LinkedIn Page, YouTube Channel, Telegram Channel.
- Отримати підтвердження права на керування профілями.
- Задати єдині бізнес-дані: назва, сайт, логотип, колірні параметри.

## Попередні вимоги
- Браузер зі збереженим станом (профіль, cookie) для упрямляння реєстрації без CAPTCHA/блокувань: див. `docs/social-browser-setup-workarounds.md`.
- Список рекомендованих доменів в `docs/state-of-project.md`.
- Перевірка наявності пакунка для конфігурації: `.env` (параметри соцмереж, якщо використовуються).

## Крок 1: Перевірка існуючих профілів
- Переглянути `docs/social-profiles.md` та `docs/social-setup-status.md`.
- Для кожної платформи вказати:
  - статус (`створено`, `підтверджено`, `очікується`);
  - URL профілю/сторінки;
  - доступні права (`власник`, `редактор`, `читач`);
  - залишковий вхід/веріфікаційні потреби.

## Крок 2: Підготовка прев’ю/медіа
- Використовувати згенеровані прев’ю `/assets/images/og/*.png`.
- Для аватарів підійти добірка 1200x630 або кадрування квадрата.
- Для банерів підійти формати Facebook/LinkedIn cover + Telegram thumbnail.

## Крок 3: Порядок створення/відновлення профілів
1. X/Twitter
   - Чи є акаунт бізнес-типу.
   - Підтвердити сайт через Settings -> Website.
   - Увімкнути додаток для керування, якщо є.
2. Facebook Page
   - Створити сторінку бізнесу.
   - Підключити Instagram Business (один аккаунт).
3. Instagram Business
   - Перемкнути профіль на Business.
   - Прив’язати до Facebook Page та додати сайт.
4. LinkedIn Page
   - Створити сторінку організації або використати існуючий акаунт.
5. YouTube Channel
   - Створити канал під ім’ям бізнесу.
6. Telegram
   - Офіційний канал +, якщо потрібен, груповий чат.

## Крок 4: Документування доступу
- Додати до `docs/social-profiles.md`:
  - посилання;
  - тип доступу;
  - нотатки з реєстрації;
  - у conveє всю інформацію про логін/пароль у зашифровану схему, демо-креденціали в вотермарк-зображення не зберігати.

## Крок 5: Перевірка та підтвердження
- Відкрити кожен профіль в режимі Read-Only/Viewer, щоб переконатися, що він публічний.
-Виконати чек-лист з `docs/social-previews-checklist.md`.

## Крок 6: Онбординг/автоматизація
- Підключити вміст:
  - планування публікацій (1–2 пости на тиждень);
  - cross-post через RSS-канал статей;
  - моніторинг коментарів/подій через менеджер або нотифікації.
- Оновити `docs/social-profile-runbook.md` параметрами під конкретні ресурси проєкту.

## Відповідальність/Терміни
- Відповідальна особа: команда проєкту / оператор.
- Коригувати план, якщо платформи вимагають ідентифікацію або документів.
- Термін: підготувати до першої публічної анонси в соцмережах.
