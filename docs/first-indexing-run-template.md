# Шаблон виконання першої перевірки індексації

Цей документ є стабільним контрактом першого запуску перевірок індексації.
Тут фіксовано передумови, команду, очікувані стани `docs/indexing-protocol.md` та наступні дії. Не змінюй інтерфейс цього файлу без оновлення `tasks.md`.

## 1. Передумови

Перед запуском мають бути виконано наступні умови:
- У `.env.example` присутні змінні `GSC_KEY_PATH`, `GSC_SITE_URL`, `BING_WEBMASTER_API_KEY`, `BING_SITE_URL`.
- В `.env` заповнені відповідні значення.
- Для Google: сервісний акаунт доданий як користувач у Search Console, увімкнено "Google Search Console API" та наявний файл ключа за `GSC_KEY_PATH`.
- Для Bing: створено API-ключ у Bing Webmaster для сайту.

## 2. Команда запуску

```bash
bash scripts/indexing-check.sh
```

Сценарій сам додасть/оновить заголовок журналу в `docs/indexing-protocol.md`.

## 3. Очікувані стани протоколу після запуску

- Якщо доступ не надано → у `docs/indexing-protocol.md` додається запис `PENDING_ACCESS` для відповідної системи.
- Якщо доступ надано → додається запис з тимчасовими даними `TODO: записати з API-відповіді` для Coverage/Scan та проблем.

## 4. Наступні дії

1. Отримати фактичні значення Coverage/Scan з панелей Google/Bing.
2. Замінити у `docs/indexing-protocol.md` заповнювачі `TODO` реальними даними.
3. Якщо виявлено проблеми — записати виправні дії в колонку `Дії`.
4. Для регулярних перевірок використовувати `cron/indexing-weekly.sh`.
